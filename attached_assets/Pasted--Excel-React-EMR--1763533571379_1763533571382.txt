

다음에 제공된 Excel 파일의 구조를 기반으로 **React 기반 강아지 EMR(전자 의무 기록) 스타일의 헬스 데이터 대시보드 웹앱**을 완성도 있게 설계하고 전체 코드를 생성해줘.

웹앱은 **의료 EMR UI(참고: [https://systemintegration.in/portfolio/emr-software-dashboard/](https://systemintegration.in/portfolio/emr-software-dashboard/) )**의 분위기를 참고하여 구성하며, **강아지 3D 모델(Three.js 또는 React-Three-Fiber)을 활용해 신체 부위별 검사 항목을 설명하는 기능**을 넣어줘.

또한 **한국어로 매우 상세한 README.md**를 포함해야 하고, 전체 프로젝트가 실제로 실행 가능한 구조여야 한다.

# ---------------------------------------------

# 🎯 **1. 기술 스택 요구 조건 (필수)**

### 프론트엔드

* **React + Vite**
* UI Framework: Tailwind**
* 시각화: **Recharts 또는 Plotly.js**
* 3D 모델: **React-Three-Fiber + Drei**
* 상태관리: **Zustand 또는 Redux Toolkit**

# 🎯 **2. 구현해야 하는 EMR 스타일 UI 요구사항 (필수)**

## 1) **EMR Dashboard 느낌**

아래 특징을 반드시 포함:

* 좌측 고정 사이드바
* 상단 환자 요약 바
* 카드형 통계 위젯
* 의료용 블루/화이트 톤
* 진료 기록 타임라인
* 검사 수치 경고(빨강/노랑/초록)
* 환자별 프로필 상세 페이지

## 2) **강아지 3D 인터랙션**

* React-Three-Fiber로 **강아지 전체 모델** 렌더링
* 특정 부위 클릭 시

  * 해당 부위 관련 검사 항목 목록 표시
  * 관련 질환 설명
  * 검사 결과 추세 그래프 팝업
* 예시:

  * 간 → ALT/AST 그래프
  * 신장 → Creatinine/BUN
  * 심장 → 심장음/영상 결과
  * 피부 → 문진 증상 및 피부질환 항목

## 3) **데이터 기반 EMR 탭 구조**

아래 탭을 생성:

1. **홈(요약)**
2. **동물 정보/고객 정보**
3. **방문 기록 타임라인**
4. **문진 데이터(모킹)**
5. **검사 결과(일반/영상)**
6. **3D 신체 부위별 검사 항목**
7. **사용자 정의 필터/군집 생성**
8. **데이터 다운로드/업로드**

# 🎯 **3. 데이터 처리 요구사항 (필수)**

엑셀 시트를 완전히 분석해 다음을 수행하는 코드를 만들어줘:

## 📌 1) 고객정보

* 동물번호 primary key
* 나이 계산 필드 생성
* 품종/성별 표준화

## 📌 2) 특정환자 방문기록

* 동물번호 join
* 방문 날짜 순 정렬
* 예약/접수 결측치 처리

## 📌 3) 사전문진데이터

* 템플릿 구조 → QuestionTemplate 테이블로 정규화
* mock 응답 생성 기능 포함

## 📌 4) 특정환자검사결과

* 값이 없는 템플릿이므로
  → 정상범위 기반 mock 검사값 생성

## 📌 5) 헬스데이터 항목(일반/영상)

* ExamMaster 단일 구조로 통합
* 검사명/코드/단위/정상범위/설명 정리
* 영상 항목의 정성/정량 구분

## 📌 6) 정상범위 파싱 후 상태 라벨링

* 높은(H), 낮은(L), 정상(N) 자동 계산
* 색상 로직 적용

## 📌 7) 이상치 탐지

* 3σ 또는 IQR 방식

## 📌 8) 군집 분석 기능

* 품종군/나이군
* ALT 상승군
* 신장 지표 상승군
* 사용자 정의 군집 저장 기능

## 🟦 **사용자 정의 필터/군집 페이지**
